## Architecture overview

The target keeps gitbench's defining spine intact: renderer -> preload (window.api) -> main IPC handlers -> application -> domain, with infrastructure implementing the side-effecting reads behind that boundary. Every hard rule survives unchanged — contextIsolation on / nodeIntegration off, ipcRenderer confined to src/preload/index.ts, pure domain/application TypeScript with no electron or node:* imports, nothing thrown across IPC (Result<T> envelopes, domain-to-DTO mapping at the seam), git spawned only through runGit.ts with argument arrays, diffText === "" as a valid clean state, and worktrees rendered as a flat list. The refactors do not move the architecture; they remove ceremony at the thin ends and fix renderer coupling/effect patterns that hurt local reasoning.

On the main/infrastructure side, the biggest structural change is the collapse of the single-implementation port layer. The five application ports (WorktreeReader, CommitReader, DiffReader, WorktreeSummaryReader, RepoWatcher) and the five forwarding-only use-case factories (makeListWorktrees, makeGetUncommittedDiff, makeListUnpushedCommits, makeListWorktreeSummaries, makeWatchRepository) are deleted. compositionRoot.ts now constructs the concrete Git CLI readers (and the ChokidarRepoWatcher) directly and hands them to handlers, while preserving the infra/application directional boundary — application still owns its errors (NotARepositoryError, WorktreeNotFoundError) and main still maps domain results to DTOs at the boundary. The behaviorless Worktree domain entity is removed; readers map straight to WorktreeDto unless real invariants are ever introduced (documented as removed in the target). GitCliDiffReader drops its hand-rolled offset/slice batching in favor of the shared mapWithConcurrency helper that the summary reader already uses. The watch layer renames isWorkingTreePathIgnored to containsIgnoredSegment with a documented any-segment path contract, and the trivial makeWatchRepository identity wrapper is gone.

On the renderer side, the worst cross-feature leak is fixed: changed-file navigation no longer imports diff-viewer's DiffFileModel. worktree-list now owns a neutral ChangedFileItem model (id, path{directory,name}, status, additions/deletions), and App.tsx becomes the composition boundary that maps diffModel.files -> ChangedFileItem[] before passing them to the detail sidebar. The ~300-line useWorktreeBrowser god-hook is split into useRepositoryCatalog, useSelectedWorktreeDetails, and useRepositoryWatcher (with useLatestRequest and two reducer modules backing atomic transitions), while useWorktreeBrowser remains a thin facade with the same public controller shape so App barely changes. Effect-driven derived state is removed: DiffView resets collapse state via a React key on model identity instead of a synchronous effect, and useScrollToSection becomes a command hook with no pendingScrollId-cleared-in-effect. useActiveFileScrollSpy receives an injected scrollContainerRef (created in App, attached to <main> by AppShell) instead of walking the DOM for the .app-shell__content magic class. The :global CSS overrides that silently failed in production are replaced by real component variants: SegmentedControl gains a density variant and DiffStat gains an emphasis variant. CommitFileChange/CommitFileChangeStatus collapse to one canonical definition in contracts/ipc/commits.ts (re-exported through the barrel, which also exports ERROR_CODES), and groupChangedFiles is memoized/hoisted so collapse state survives unrelated re-renders.

## Layers

### Domain (src/domain)

**Role.** Business entities and invariants in pure TypeScript. In the target this layer is effectively empty for worktrees: the behaviorless Worktree alias of WorktreeDto is removed because it protected no invariants. The layer is retained as a directory only as a home for a future entity if real worktree invariants (e.g. isDetached(), non-empty path) ever appear.

**Rules.** Imports nothing outside domain. No electron, no node:*, no React, no contracts. If reintroduced, an entity must carry actual behavior, not be a structural copy of a DTO.

### Application (src/application)

**Role.** Application-level errors only in the target: NotARepositoryError and WorktreeNotFoundError under worktrees/errors. The output ports and forwarding-only use-case factories are deleted; there is no longer any pass-through orchestration code here because this viewer has no application logic to host between a handler and a single git reader.

**Rules.** Pure TypeScript — no electron, no node:*, no React. Depends only on domain. The infra/application directional boundary is preserved conceptually (infrastructure may depend on application errors; application never depends on infrastructure), even though the per-port interface files are gone. Reintroduce a port only when a second implementation or a real test seam actually exists.

### Infrastructure (src/infrastructure)

**Role.** Output adapters: the Git CLI readers (GitCliWorktreeReader, GitCliDiffReader, GitCliWorktreeSummaryReader, GitCliCommitReader), pure parsers (porcelain, commit log, worktree summary), git helpers (runGit, refResolves, unpushedStrategy, mapWithConcurrency), git error types, and the filesystem watch (ChokidarRepoWatcher, recursiveWatch). In the target, readers are concrete classes consumed directly by main rather than implementations of an application port interface.

**Rules.** May import application (errors), domain, and node:*. Must not be imported by application or domain. All git access flows through runGit.ts with argument arrays — never an interpolated shell string. GitCliDiffReader uses mapWithConcurrency for untracked-diff batching. ChokidarRepoWatcher's containsIgnoredSegment matches any path segment against the ignored set, with the absolute/relative input contract documented.

### Contracts (src/contracts)

**Role.** The transport surface: channels, Result<T>, DTOs (repository, worktrees, worktreeSummaries, diff, commits, watch), ERROR_CODES/ErrorDto, and the DesktopApi (window.api) type. commits.ts is the single canonical home for CommitFileChange and CommitFileChangeStatus.

**Rules.** Imports nothing outside contracts. The barrel index.ts re-exports the commit types and ERROR_CODES so both renderer and main import the contract surface from one point. Channel string values and error-code string values are byte-stable parts of agent_docs/ipc-contract.md and change only alongside that doc in the same commit.

### Main (src/main)

**Role.** Electron lifecycle, composition root, IPC input adapters, and DTO/error mapping. compositionRoot.ts constructs the concrete readers and the watcher; registerHandlers.ts injects them (plus WatchController) into the per-channel handlers; handlers delegate to a reader, map the result to a DTO via mappers, and return it inside a Result<T>; watchController.ts owns the single active filesystem watch.

**Rules.** May import application, domain, infrastructure, contracts, electron, node:*. Nothing throws across IPC — handle()/handleWithEvent() wrap every outcome in Result<T> and errorMapper classifies failures into ErrorDto codes. Domain/reader results are mapped to DTOs at the boundary; raw entities never cross IPC. compositionRoot.ts is the only place concrete adapters are constructed.

### Preload (src/preload)

**Role.** The single ipcRenderer bridge exposing window.api to the renderer.

**Rules.** The only file in the codebase that imports ipcRenderer. Imports contracts and electron only. contextIsolation stays true and nodeIntegration stays false.

### Renderer (src/renderer/src)

**Role.** Feature-based React UI following app -> features -> shared. app/App.tsx composes features and is the composition boundary mapping diffModel.files to ChangedFileItem[]; features own domain-specific rendering and behavior (worktree-browser orchestration hooks, worktree-list, diff-viewer, sidebars, welcome); shared holds reusable UI primitives (Button, Badge, DiffStat with emphasis variant, SegmentedControl with density variant), the desktopApi IPC bridge, preferences, and generic utilities.

**Rules.** app -> features -> shared, one direction. shared must not import from features or app and must not import IPC DTOs or feature models. Features must not import another feature's private files; cross-feature use goes through a public index.ts. All IPC access flows through shared/api/desktopApi.ts, which unwraps Result<T> and throws ApiError on failure. Changed-file navigation imports the worktree-list-owned ChangedFileItem, never diff-viewer's DiffFileModel.

## Key data flows

### Pick repository and list worktrees

App.tsx (Welcome onOpenRepository) -> useWorktreeBrowser facade -> useRepositoryCatalog.openRepository -> desktopApi.pickRepo (repo:pick) then desktopApi.listWorktrees (worktrees:list) -> preload window.api -> main registerHandlers -> pickRepositoryHandler (Electron dialog, bypasses application) and listWorktreesHandler -> compositionRoot's GitCliWorktreeReader.listWorktrees -> runGit('worktree list --porcelain') -> porcelainParser -> worktreeMapper -> WorktreeDto[] in Result<T> -> desktopApi unwraps -> repositoryCatalogState reducer commits the new repo transactionally (kept unchanged on failure) -> App renders RepositorySidebar with the flat WorktreeList.

### Load summaries for all worktrees

useRepositoryCatalog (after worktrees commit) -> useLatestRequest guards a desktopApi.listWorktreeSummaries (worktrees:summaries) call -> listWorktreeSummariesHandler -> GitCliWorktreeSummaryReader (mapWithConcurrency over paths; per-worktree failures dropped) -> runGit status/numstat -> worktreeSummaryParser -> worktreeSummaryMapper -> WorktreeSummaryDto[] -> reducer stores summaries; rows stay selectable if this secondary load fails.

### Select worktree -> diff + unpushed commits

WorktreeRow onSelect -> App.selectWorktree (startTransition) -> useWorktreeBrowser -> useSelectedWorktreeDetails.selectWorktree -> selectedWorktreeState reducer commits selectedPath and clears old details, then fires desktopApi.getDiff (diff:get) and desktopApi.listUnpushedCommits (commits:unpushed) under independent useLatestRequest guards -> getDiffHandler -> GitCliDiffReader.getUncommittedDiff (assertHeadExists, git diff HEAD, untracked --no-index via mapWithConcurrency) returns unified diff text ("" is valid clean state); listUnpushedCommitsHandler -> GitCliCommitReader (unpushedStrategy, commitLogParser) -> commitMapper -> ListUnpushedCommitsResponse -> App builds diffModel via buildDiffModel and maps diffModel.files -> ChangedFileItem[] for WorktreeDetailSidebar; Workspace renders DiffView.

### File-watch auto-refresh

useRepositoryWatcher (on worktree-root set change) -> desktopApi.startWatch (watch:start) -> watchHandlers -> WatchController starts ChokidarRepoWatcher (FSEvents on macOS / Chokidar elsewhere, debounced, containsIgnoredSegment filtering) -> on debounced change main pushes repo:changed back to the requesting renderer -> desktopApi.onRepoChanged subscription in useRepositoryWatcher -> quietly reloads the worktree list and re-loads diff/commits for the current selection (no loading spinner; diff swapped only when text changed). Watch restarts only when the root set changes and stops on unmount via watch:stop.

### Sidebar file selection -> reveal + scroll in diff

WorktreeDetailSidebar (ChangedFilesSection over memoized grouped ChangedFileItem[]; FileListProvider holds collapse state using the neutral model, no DiffFileModel) -> onSelectFile -> App navigation.selectFile -> useDiffNavigation emits a reveal-and-scroll navigationTarget (no synchronous effect reset) -> DiffView (keyed by model identity) un-collapses the target file and useScrollToSection command hook scrolls it into view; useActiveFileScrollSpy, fed the App-injected scrollContainerRef attached to <main> by AppShell, reports the active file back via onActiveFileChange.

## Cross-cutting concerns

### Nothing throws across IPC

handle()/handleWithEvent() in src/main/ipc/handle.ts wrap every handler outcome in Result<T>; errorMapper.ts classifies application/infrastructure errors into ErrorDto codes; desktopApi.ts unwraps and rethrows as ApiError on the renderer side. Unchanged by the refactors — even with ports gone, handlers still delegate -> map -> return Result.

### Domain entities never cross the wire

The mappers/ seam (worktreeMapper, worktreeSummaryMapper, commitMapper, errorMapper) still maps reader results to DTOs at the boundary. With the Worktree entity removed, readers produce WorktreeDto-shaped data and the mapper remains as the compile-time tripwire so a divergence is a build error.

### Git spawning discipline

runGit.ts stays the single choke point: argument arrays only, stable env, stderr classified into NotARepositoryError/WorktreeNotFoundError/GitNotInstalledError/GitCommandFailedError, maxBuffer for large diffs. GitCliDiffReader's switch to mapWithConcurrency changes batching, not the spawn contract; no path is ever interpolated into a shell string.

### Context isolation and ipcRenderer confinement

contextIsolation: true and nodeIntegration: false remain in createWindow.ts; ipcRenderer still appears only in src/preload/index.ts exposing window.api. Renderer reaches IPC solely through shared/api/desktopApi.ts. None of the refactors touch this.

### Pure domain/application layers

What remains of application (the two error classes) stays pure TypeScript with no electron/node:* imports. Deleting the ports does not introduce framework imports into application; the concrete readers that import node:* live in infrastructure and are constructed only in compositionRoot.ts.

### Clean-worktree and flat-list invariants

diffText === "" remains a valid success state, handled in App (isCleanWorktree) and DiffView's clean branch; GitCliDiffReader still refuses to fake an empty diff on an unborn HEAD. Worktrees still render as a flat WorktreeList; tree UI is reserved for changed files within a worktree, which now operate over the neutral ChangedFileItem model.

### Race-condition / latest-wins freshness

AbortController-based freshness (now centralized in useLatestRequest and consumed by useRepositoryCatalog/useSelectedWorktreeDetails) preserves the independent latest-wins guards for summaries, diff, and commits, plus repository-replacement invalidation of in-flight detail requests. IPC invokes are not cancellable; these remain freshness guards only.

### Auto-refresh without flicker

useRepositoryWatcher keeps the watcher restarting only when the worktree-root set changes and stopping on unmount; the repo:changed subscription silently reloads list + selected diff/commits without flipping loading flags, swapping diff text only when it changes to preserve scroll position. Behavior is preserved across the hook split.

### Contract-doc coupling

Channel names and error-code strings in contracts/ipc remain byte-stable and tied to agent_docs/ipc-contract.md; any channel/DTO/error-code change must update that doc in the same commit. The refactors do not add or rename channels — repo:pick, worktrees:list, worktrees:summaries, diff:get, commits:unpushed, watch:start, watch:stop, repo:changed are unchanged.
